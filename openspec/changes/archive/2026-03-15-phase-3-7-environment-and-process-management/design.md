## Context

Niuma 当前已经实现了文件系统、Shell、Web、Git、网络、数据处理和加密解密工具，但缺乏环境变量和进程管理的基础功能。这些工具对于自动化运维、系统监控和环境配置是必要的。

**当前状态：**
- 已有 26 个内置工具
- 工具框架已完善（Tool 抽象类 + Registry 模式）
- 测试框架已就绪（vitest）

**约束：**
- 遵循 MCP 优先架构，但环境变量和进程管理是基础系统功能，应内置实现
- 必须保持工具的安全性和稳定性
- 需要完整的错误处理和中文错误消息

## Goals / Non-Goals

**Goals:**
- 实现环境变量读取和设置功能（env_get、env_set）
- 实现进程列表查询和终止功能（process_list、process_kill）
- 提供完整的参数验证和错误处理
- 保持与现有工具框架的一致性
- 100% 测试覆盖

**Non-Goals:**
- 不支持跨进程环境变量设置（仅当前进程）
- 不支持进程创建（使用现有的 exec 工具）
- 不支持高级进程管理（如优先级调整、资源限制）

## Decisions

### 1. 技术选型：Node.js 内置模块 + ps-tree

**选择：**
- 环境变量：使用 `process.env`（Node.js 内置）
- 进程管理：使用 `ps-tree` 库（跨平台进程树遍历）

**理由：**
- `process.env` 是 Node.js 标准 API，零依赖
- `ps-tree` 是成熟的跨平台库，支持 Linux、macOS 和 Windows
- 避免调用系统命令（如 `ps`、`kill`），保证跨平台兼容性

**替代方案考虑：**
- 调用系统命令（`ps`、`kill`）：跨平台兼容性差，需要大量条件判断
- 使用 `node-pty`：过于复杂，主要用于终端模拟

### 2. env_set 只支持当前进程

**选择：**
- `env_set` 只修改当前进程的环境变量，不影响子进程或父进程

**理由：**
- Node.js 环境变量作用域限制
- 安全性考虑：避免误修改系统环境
- 符合 Unix 哲学：最小权限原则

### 3. process_kill 支持多信号和强制终止

**选择：**
- 支持标准终止信号（SIGTERM）
- 支持强制终止（SIGKILL）
- 提供进程树终止选项

**理由：**
- 灵活性：适应不同场景
- 安全性：优先使用 SIGTERM，失败时回退到 SIGKILL
- 完整性：支持终止进程树（包括子进程）

### 4. 工具命名和参数设计

**选择：**
- 工具名：`env_get`、`env_set`、`process_list`、`process_kill`
- 参数设计：简洁明确，遵循现有工具命名规范

**理由：**
- 一致性：与现有工具命名风格一致（如 `file_search`、`git_status`）
- 可读性：工具名自解释，参数清晰

## Risks / Trade-offs

### Risk 1: 进程终止可能影响系统稳定性

**风险：** 错误终止关键进程可能导致系统不稳定

**缓解措施：**
- 添加受保护进程列表（如 init、kernel 进程）
- 添加确认机制（confirm 参数）
- 提供进程信息预览（名称、命令行参数）
- 记录终止操作日志

### Risk 2: 环境变量设置可能影响其他工具

**风险：** 修改环境变量可能影响后续工具调用

**缓解措施：**
- 明确文档说明：env_set 只影响当前进程
- 建议用户在子进程中使用，避免污染主进程
- 提供 env_get 功能，方便查询当前环境

### Risk 3: ps-tree 依赖的跨平台兼容性

**风险：** ps-tree 在某些平台可能不稳定

**缓解措施：**
- 添加充分的错误处理
- 提供平台检测和降级方案
- 添加单元测试覆盖所有支持平台

### Trade-off: 功能完整性 vs 实现复杂度

**权衡：** 不支持高级进程管理功能（如优先级调整、资源限制）

**理由：**
- 高级功能使用频次低
- 可以通过 MCP Server 提供
- 保持核心轻量级

## Migration Plan

**部署步骤：**
1. 安装新依赖：`pnpm add ps-tree @types/ps-tree`
2. 创建 `niuma/agent/tools/system.ts`
3. 修改 `niuma/agent/tools/registry.ts` 注册新工具
4. 创建测试文件 `niuma/__tests__/system-tools.test.ts`
5. 运行测试验证
6. 更新工具数量文档

**回滚策略：**
- 从 registry.ts 移除工具注册
- 删除 system.ts 文件
- 回退依赖版本

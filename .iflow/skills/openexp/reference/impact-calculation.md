# 影响力计算逻辑

## 概述

影响力用于衡量一条经验在经验库中的重要程度，综合考虑使用频率、可靠性和关联关系。

## 计算公式

### 原始公式
```
影响力 = usage_count * 5 + confidence * 50
```

### 优化公式（考虑关联关系）
```
影响力 = usage_count * 5 + confidence * 50 + backlinks * 10 + referenced_weight
```

**说明**：
- `usage_count * 5` - 使用频率贡献
- `confidence * 50` - 可靠性贡献
- `backlinks * 10` - 被引用程度贡献
- `referenced_weight` - 引用的重要经验的权重总和

其中：
- `backlinks` - 被其他经验引用的次数
- `referenced_weight` - 引用的重要经验的权重总和

### referenced_weight 计算

```
referenced_weight = Σ (被引用经验的影响力 / 被引用经验的引用数)
```

**解释**：
- 如果一条经验引用了另一条经验，会获得被引用经验的影响力份额
- 份额 = 被引用经验的影响力 / 被引用经验被引用的总次数
- 这样可以避免循环引用导致的影响力无限增长

## 示例

### 示例 1：无关联关系

```
经验 A：
- usage_count: 10
- confidence: 0.8
- backlinks: 0
- referenced_weight: 0

影响力 = 10 * 5 + 0.8 * 50 + 0 * 10 + 0 = 50 + 40 + 0 + 0 = 90
```

### 示例 2：有被引用

```
经验 A：
- usage_count: 10
- confidence: 0.8
- backlinks: 3（被经验 B、C、D 引用）
- referenced_weight: 0

影响力 = 10 * 5 + 0.8 * 50 + 3 * 10 + 0 = 50 + 40 + 30 + 0 = 120
```

### 示例 3：引用重要经验

```
经验 A：
- usage_count: 5
- confidence: 0.7
- backlinks: 0
- referenced_weight: 45（引用了经验 E，E 的影响力为 90，被引用了 2 次）

经验 E：
- usage_count: 10
- confidence: 0.8
- backlinks: 2（被经验 A、F 引用）
- referenced_weight: 0

影响力(A) = 5 * 5 + 0.7 * 50 + 0 * 10 + 45 = 25 + 35 + 0 + 45 = 105
影响力(E) = 10 * 5 + 0.8 * 50 + 2 * 10 + 0 = 50 + 40 + 20 + 0 = 110

referenced_weight(A) = 90 / 2 = 45
```

## 权重说明

| 因素 | 权重 | 说明 |
|------|------|------|
| usage_count | 5 | 每次使用增加 5 分，反映实际使用频率 |
| confidence | 50 | 满置信度增加 50 分，反映可靠性 |
| backlinks | 10 | 每个反向链接增加 10 分，反映被引用程度 |
| referenced_weight | 动态 | 获得被引用经验的影响力份额，反映关联价值 |

## 迭代计算

由于 `referenced_weight` 依赖于其他经验的影响力，影响力需要迭代计算：

1. **初始计算**：使用原始公式计算初始影响力
2. **迭代更新**：根据新的影响力重新计算 `referenced_weight`
3. **收敛判断**：当所有经验的影响力变化小于阈值（如 1 分）时停止
4. **最大迭代次数**：防止无限循环，通常 10 次迭代足够收敛

## 应用场景

### 排序和展示
- 在索引地图中按影响力降序展示经验
- 限制每个维度只展示影响力最大的前 20 条

### 经验清理
- 定期清理影响力低于阈值（如 30 分）的过时经验
- 但要保留最近创建的经验（如 30 天内），无论影响力如何

### 经验推荐
- 优先推荐高影响力的经验给用户
- 在应用经验时，优先使用高影响力的方案

## 注意事项

1. **避免循环引用**
   - 检测循环引用并打断
   - 使用最大迭代次数限制

2. **新经验保护**
   - 新创建的经验（如 30 天内）即使影响力低也不删除
   - 给予新经验一个最小影响力保障（如 30 分）

3. **定期重新计算**
   - 每周重新计算所有经验的影响力
   - 或在经验使用/更新时触发增量计算

4. **透明度**
   - 在经验 frontmatter 中记录影响力计算的各个组成部分
   - 方便用户理解影响力来源

## Frontmatter 字段

```yaml
---
id: exp_preference_20260311_140000_1
type: preference
confidence: 0.8
usage_count: 10
impact_score: 120          # 总影响力
backlinks: 3               # 反向链接数
referenced_weight: 0       # 引用权重
last_impact_update: 2026-03-11T23:00:00Z
---
```
